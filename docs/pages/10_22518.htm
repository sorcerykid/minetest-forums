The minetest package on my distro and probably others too is configured to allow running multiple instances of minetest with different minetest@[instance].service files.

Is it possible to have different mapgen settings in these instances? Because as I understand it, minetest.conf is a global config file for all worlds.

Re: per-instance mapgen settings

by rubenwardy » Tue Apr 09, 2019 4:51 pm

The minetest.conf mapgen setting just set the default settings when creating a world. The settings are then fixed for the world once created. Ie: changing mapgen settings in minetest.conf won't affect existing worlds.

The mapgen settings for a particular world can be found in world.mt
