Re: what is the latest version of minetest that works on a p

by Krock » Mon Apr 22, 2019 4:32 pm

Yes, any recent version of Minetest will run on ARM devices.

If you cannot find 5.0.1 binaries for your ARM-based RPI, then it's still possible to build it from the source.
https://github.com/minetest/minetest/bl ... #compiling
You probably will only need the irrlicht, sqlite3 and zlib libraries if it's going to be a server-only build (CMake flag `-DBUILD_CLIENT=FALSE`).

Re: what is the latest version of minetest that works on a p

by dawgdoc » Wed Apr 24, 2019 8:20 am

What OS are you running on your RPi?

If you are running the latest Raspbian, it is based on Debian Stretch. The Minetest version in the Stretch repositories is 0.4.15. You can get to Minetest 0.4.17.1 by using the package from the stretch-backports repositories. The only/best way to get 5.01 on Raspbian, I think, would be to do as Krock stated and build from source.

If you are running Ubuntu Mate built for the RPi, the latest version is based on Ubuntu 18.04.2. If this is the case you can install Minetest 5.0.1 from the stable PPA. You can use the same source for some older versions of Ubuntu Mate for RPi. Again, you could build from source.
